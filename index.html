How I work

Build around the workflow people actually use.

Good automation does not start with a flashy tool. It starts with understanding the current process, the exceptions, the risks, and the decisions staff still need to control.

Map the work

Identify tasks, documents, handoffs, delays, and avoidable errors.

Simplify the process

Automate carefully

Measure and improve

Create reporting and checks that make the workflow easier to trust.

Featured project

filepackR: document processing and Intelligent File Routing.

One example of the type of operational system I build: software that turns fragile manual file sorting into a cleaner and more repeatable workflow.

From mixed PDF packets to organized records.

filepackR is PDF splitter, merger, and document-processing software built to identify, organize, and route files through the right workflows. It combines OCR-powered classification, customizable rules, review queues, and on-device processing.

  • Split mixed PDF packets into separate documents.
  • Merge related pages and supporting records.
  • Apply consistent filenames and routing logic.
  • Surface exceptions for human review.
